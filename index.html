Overview

About the Project

Secure File Sharing and Recovery System is proposed as a platform that helps users upload, organize, share, restore, and monitor important files in a protected environment. The project addresses two connected needs: secure file access and dependable recovery after deletion, accidental loss, or version conflicts.

Following the structure seen in the sample PRD, URS, TDM, slides, and user manual, this website presents the project through its problem statement, objectives, user roles, system features, and technical direction.

Research

Problem Statement and Objectives

Problem Statement

Existing file sharing tools often prioritize convenience, but users still face risks such as unauthorized access, accidental deletion, weak recovery flows, and limited visibility into who accessed or changed a file.

Project Objective

Build a secure platform that supports protected upload and sharing, recovery options for lost files, and clear access control for different types of users.

Expected Outcome

Deliver a usable web-based system that combines encryption-minded design, recovery workflows, activity tracking, and role-based management in one coherent solution.

Users

Target Users and Roles

General User

File Owner

Uploads files, controls sharing permissions, restores previous versions, and reviews file activity.

Collaborator

Authorized Recipient

Receives access to shared files, downloads permitted items, and collaborates within assigned privileges.

Administrator

System Manager

Oversees platform users, monitors logs, manages recovery cases, and enforces security settings.

Features

Core System Features

Secure Upload and Storage

Files are stored with strong protection in mind, with authentication and access boundaries applied before sharing.

Controlled File Sharing

Users can share files through permission-based access, limiting who can view, download, or manage documents.

Recovery and Version Restore

Deleted or changed files can be recovered through recycle, backup, or historical version restoration workflows.

Audit Trail

The system records important actions such as uploads, downloads, sharing changes, and recovery events for traceability.

Role-Based Access Control

Different roles operate with different privileges to reduce unauthorized exposure and improve governance.

Dashboard and Alerts

Users and administrators receive visibility into file status, access events, and recovery notifications.

Design

Technical Direction

Based on the sample Technical Design Manual and User Requirement Specification, the system can be organized into a web interface, an authentication and access-control layer, secure file storage, a recovery module, and an administrative monitoring panel.

Non-functional requirements for this project should emphasize security, reliability, maintainability, usability, and scalability. These areas were consistently highlighted across the provided sample FYP documents and fit this project especially well.
